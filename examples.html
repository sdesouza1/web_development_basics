HyperText Markup Language

You can view the source of any webpage using the "view source" tool available in all major desktop browsers via the key combination Ctl-U.
A markup language is a modern system for annotating a document in a way that is syntactically distinguishable from the text. So, HTML is a formatting and layout language for text as well as user interface elements (input boxes, menus, etc.) in documents that contain hyperlinks. HTML uses pairs of tags to specify elements. For example, all HTML documents start with an <html> tag and end with an </html> tag. Comments start with <!-- and end with -->. Comments can contain other tags, which will be ignored until the --> is reached.

Cascading Style Sheets

The source code for stylesheets is also available by clicking the link in the HTML source.
Cascading Style Sheets (CSS) is a style sheet language used for describing the look and formatting of a document written in a markup language. They are called "cascading" because HTML elements inherit all style attributes from their parents. This inherited style can be overridden by local style attributes. Using styles, you can control everything from colors and font properties to layout and even some dynamic behavior.

Javascript/JQuery

Javascript is the programming language of the browser. We've seen AJAX, DOM manipulation. Javascript also supports objects.
function counter() {
  var count = 0;
  this.count_one = function() { count++; };
  this.get_count = function() { return count; };
}

function test_counter() {
  var c = counter_object;
  var success = 1;
  for (var i = 0; i < 10; i++) {
    if (c.get_count() != i) {
      alert("counter is broken!");
      success = 0;
      break;
    }
    c.count_one();
  }

  if (success) {
    alert("counter passed test");
  }
}


Most of the time, javascript developers lean heavily on open source libraries such which simplify everything from network requests (AJAX), changing the current page (DOM Manipulation), or even animations.
$("#hidden_div").slideToggle();


Prototype is another JS library with similar functionality to JQuery, including the use of $ and CSS selectors.

Javascript is also useful for validating user input before sending it to the server. However, the server should never rely on client-side validation of input, and should also perform it's own validation. Javascript can help you avoid unnecessary server requests though.

In addition to automated users, websites must be designed to be used with screen reading and voice command software so that it is accessible to as many people as possible. Additionally, website visitors are increasingly on mobile devices with smaller screens. This must also be taken into consideration, for example by using percentages rather than absolute pixel values.

Git

Git is a free and open source distributed version control system designed to handle everything from small to very large projects with speed and efficiency. Git is easy to learn and has a tiny footprint with lightning fast performance.
> git clone git://github.com/your_user_name/web_development_basics.git
> git status
> git add new_or_modified_file
> git commit -m 'This is the commit message, if you leave off the minus m, it will open a text editor for you to write your commit log. It will use whatever the EDITOR variable is set to. Be careful with your apostrophes/quotes, since contractions can mess it up unles you use ""'
> git rm file_to_be_removed
> git mv file_or_files_to_be_moved target_location
# to create a branch starting from current code
> git checkout -b branch_name
# to switch to an existing branch (master is the default branch)
> git checkout master
> git push
> git pull
> git log
> git blame
# oops > git commit --amend > git checkout filename
> git reset HEAD filename
For more examples and to learn how to use the more powerful features of git, see the Git Book.

Dropbox/Google Drive/Backups

There are several other options for sharing files among teams. As part of your google account via mix, you have access to google drive. You can use this to share files, and Google Documents, which can be collaboratively edited by multiple users simultaneously. Dropbox(disclaimer: referral link) is also a good option for sharing files. It syncs all files locally to each of the computers that you install it on. Both Google Drive and Dropbox have some limited version control abilities, but both do well with documents that have binary formats, especially large ones. None of these solutions are failsafe backups, and it is usually best to sync important documents and files to as many systems as you can spare the disk space on. Source code in particular is highly valuable and takes up very little space. It is also highly compressible. The nice part about Git and other distributed revision control software is that every copy of the repository has the full project history by default.
